В данном примере показано, как можно реализовать кастомный фильтр путем модификации текста запроса в обработчике события ValueChangeEvent поля ввода.

См. метод reload() класса CustomFilterQuerySample. Если поле фильтра не пустое, данный метод добавляет в запрос условие where и устанавливает соответствующее значение параметра запроса. Значение параметра содержит префикс (?i), что обеспечивает регистро-независимый поиск.

Фасет DataLoadCoordinator в данном примере не используется. Вместо него метод reload() вызывается явно в обработчике события BeforeShowEvent.

Обратите внимание, что компонент универсальный фильтр можно использовать совместно с кастомным фильтром.
